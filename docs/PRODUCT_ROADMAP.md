## 1. 背景と診断（なぜ作り直すか）

- SNSフォロワー約12万人に対して**有料会員 0 人**。壁紙は外部の STORES では毎日売れている → 需要はあるが**自サイトの体験が価値を伝えられていない**。
- 原因の診断（2026-07-02 コード横断調査に基づく）:
  1. **概念の乱立**: WHATIF Gallery / IMAGINE / The Club / STORE の4ブランド＋「エピソード・テンプレート・バナー・壁紙・club item」の語彙が混在。「$3/月で何が手に入るか」を1文で言えない。
  2. **購入にログイン必須**: 壁紙単品購入は anonymous → まずログイン。STORES はゲスト購入できる。売れる場所と売れない場所の最大の差。
  3. **無料と有料の線引きが崩壊**: エディタはゲストでもフル解像度PNGを透かしなしでエクスポート可能（`BannerEditor.tsx` の `handleExport` に権限分岐なし）。商品の核が「クレジットなしのクリーン画像」なのに、無料で実質クリーン相当が手に入る恐れ → premium の固有価値が消える。

## 2. To-Be（生まれ変わり後の姿）

**1つのブランド・1つのドメイン（`whatif-ep.xyz`）・1つのアプリ。**
作品ページをハブに「見る → 無料DL → 編集 → 保存(ログイン) → クリーン/壁紙(課金)」の一本のファネル。

```
WHATIF
├─ Works（ギャラリー＝玄関）
├─ 作品ページ＝ハブ: 無料DL / この絵を編集 / 壁紙を買う / Premium
├─ Edit（エディタ。ログイン不要で全機能お試し）
├─ My Designs（保存＝無料アカウントの価値）
├─ Premium（IMAGINE premium を本ドメインに統合した単一プラン）
└─ (footer) The Club  ← legacy。下記 §4
```

## 3. 権限ラダー（Phase 0 決定・2026-07-02 確定）

| 層 | できること | 設計意図 |
|---|---|---|
| **ゲスト（ログインなし）** | 閲覧 / クレジット付き画像DL / エディタ全機能 / **クレジット付き**エクスポート | 摩擦ゼロで「触って楽しい」まで到達させる |
| **無料アカウント** | ＋デザインのクラウド保存（複数）/ My Designs / 自分の画像アップロード | 「保存したい」瞬間がログインのトリガー |
| **Premium（月額）** | ＋全壁紙パック / **クリーン（クレジットなし）エクスポート** / プレミアムアセット・テンプレート | 「クリーンなものは全部使い放題」と1文で言える価値 |
| **単品購入** | その作品の壁紙パックのみ。**ログイン不要のゲスト購入**（メールにDLリンク送付） | STORES で実証済みの衝動買い需要を自サイトで受ける |

決定事項:

- **クレジットポリシー**: ゲスト/無料＝クレジット合成付きエクスポート、Premium＝クリーン。現行の「誰でも透かしなしフル解像度」は廃止する。
- **保存の線引き**: 現行 IMAGINE の「ゲスト localStorage 1件」は維持してよいが、クラウド保存＝無料アカウントの価値とする。
- **DB再構築**: 有料会員 0 のため**破壊的な再構築も許容**（実施時はユーザーに相談のうえ SQL 手動実行）。
- 価格（Premium 月額・壁紙単品）は Phase 2 までに STORES 実売価格と整合させて最終決定（未確定）。

## 4. The Club の扱い（決定・2026-07-02）

- **廃止ではなく「フッターリンクへの格下げ＋隔離」**。
- 理由: **Instagram のサブスクリプション特典**として「The Club の壁紙DL」を提供しているため、受け皿として残す必要がある。
- 方針: 新しいプロダクト構造（Premium/壁紙/エディタ）とは**分離した legacy 資産（負債）**として扱う。発展させない。新機能から参照しない。いずれ Instagram 特典の整理とともに廃止する。
- 実装上: ヘッダーのハンバーガーメニューから外し、フッターのリンク1つに格下げ。legacy 会員ログイン（`/auth/legacy-login`）と `club_items` ダウンロードは現状維持。

## 5. フェーズ計画

### Phase 0: 価値と価格の確定 — **完了（本書 §3–4）**

残検証（starter テンプレのクレジット）**検証済み・2026-07-02**:

- クレジットは画像への焼き込みではなく、**バナー内の削除可能なテキスト要素2つ**（タイトル `/IMAGINE: EP{code}` + 宣伝文）。`productionProjects.ts` の `buildInstagramFeedTextElements` が feed バナーのみに付与し、テンプレ昇格時は `elements` を無加工コピー。
- よってゲストはクレジット要素を削除してクリーン画像をエクスポート**できる**。ただし出力は feed の **1080×1350（4:5）のみ**。有料壁紙（1440×2560 等の 9:16/16:9）とは解像度・アスペクト比が別物で、完全代替にはならない。壁紙サイズのテンプレは `premium` 昇格かつ Gallery 非露出でゲート済み。
- 判定: **即時の対処は不要**。Phase 3 の「ゲスト/無料＝エクスポート時クレジット強制合成」で構造的に塞ぐ（エレメント削除に依存しない合成方式にすること）。

### Phase 1: 技術統合（= CONSOLIDATION_PLAN M1〜M6）

単一 Next.js アプリ化。BannerEditor 移植 → `@supabase/ssr` 一本化・SSO撤去 → アセット key 方式化 → 周辺ページ統合 → データ/URL移行 → 旧 IMAGINE 凍結。
**全 UX 改修の前提**（先にやらないと以降の改修が2サイト二重実装になる）。

> 先行クイックウィン（統合を待たない）: **壁紙のゲスト購入**は Gallery 単独で実装可能。売上検証を先に始められる。

### Phase 2: IA・ファネル再構築

- ナビと語彙の統合（The Club をフッターへ、Premium ページを本ドメインに新設）
- 作品ハブページ再設計（無料DL / 編集 / 壁紙 / Premium を優先順位付きで1画面に）
- ゲスト編集の磨き込み（Gallery からワンタップ編集開始、保存時に「アカウント作成で保存できます」）
- 単品購入のゲストチェックアウト（Phase 1 中に先行済みならスキップ）
- 価格の最終決定・反映

### Phase 3: Premium 価値の実装

- エクスポートのクレジット合成分岐（ゲスト/無料＝クレジット付き、Premium＝クリーン）
- プレミアムアセットライブラリ（`default_images`）のエディタ内可視化（ロック付きで見せてタップでアップセル）
- 壁紙パックと「編集した自分のデザイン」の統合的な見せ方

### Phase 4: グロースループ

- SNS → 作品ディープリンク運用（OGP は Next 母体の強み）
- 投稿ごとに「この絵を編集する」URL を添付
- 計測: ゲスト編集開始率 / 保存→登録転換率 / 登録→Premium転換率 / 作品ページ→壁紙購入率
